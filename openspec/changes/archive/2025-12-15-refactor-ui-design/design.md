# 设计方案: UI 重构

## Context
**打字小达人**是一个面向7岁儿童的打字练习应用，需要通过视觉设计来保持儿童的学习兴趣。当前设计较为基础，需要升级为更具吸引力和个性化的设计。

### 约束条件
- 纯 CSS 实现（无 UI 框架依赖）
- 保持现有功能不变
- 确保可访问性和可读性
- 性能优化（避免过多动画导致卡顿）

## Goals / Non-Goals

### Goals
- 创造独特、令人难忘的视觉风格（避免 AI 通用美学）
- 使用适合儿童的 playful/toy-like 美学方向
- 增强视觉层次和信息传达
- 添加有趣的动画和交互反馈
- 提升整体用户体验

### Non-Goals
- 不改变现有业务逻辑
- 不添加新功能
- 不使用第三方 UI 组件库
- 不追求过度复杂的动画效果

## Decisions

### 1. 美学方向: Playful Candy Land
选择 **糖果乐园 (Playful Candy Land)** 风格，特点：
- 圆润、柔和的形状
- 明亮的糖果色系
- 有趣的弹跳动画
- 温暖、友好的氛围

**理由**: 最适合7岁儿童的年龄特点，既不会过于幼稚，又保持趣味性。

### 2. 字体选择
- **标题字体**: Baloo 2 (Google Fonts) - 圆润、友好、适合儿童
- **正文字体**: Quicksand - 清晰易读、现代感
- **键盘显示**: Fredoka - 粗壮、活泼

**理由**: 避免使用 Inter、Roboto 等通用字体，选择更有个性且适合儿童阅读的字体。

### 3. 配色方案
```css
/* 主色调 - 彩虹糖色系 */
--candy-pink: #FF6B9D;
--candy-orange: #FF9F43;
--candy-yellow: #FECA57;
--candy-green: #5CD859;
--candy-blue: #54A0FF;
--candy-purple: #9C88FF;

/* 背景 */
--bg-gradient: linear-gradient(135deg, #FFF5F8 0%, #F0F8FF 50%, #FFF8F0 100%);

/* 强调色保持现有手指颜色不变 */
```

### 4. 动画策略
优先使用 CSS 动画，关键动画点：
- 页面加载：错落入场动画 (staggered reveal)
- 按钮悬停：弹跳缩放效果
- 成功反馈：星星爆发/纸屑效果
- 键盘按键：3D 按压效果
- 背景：缓慢浮动的几何图形

**性能考虑**:
- 使用 `transform` 和 `opacity` 进行动画
- 限制同时运行的动画数量
- 使用 `will-change` 提示浏览器

### 5. 组件设计规范

#### 卡片
- 更大的圆角 (20px+)
- 柔和的阴影
- 悬停时轻微浮起 + 阴影扩展

#### 按钮
- 圆润的药丸形状
- 渐变背景
- 按压时有弹性效果
- 添加微光效果

#### 徽章/标签
- 使用胶囊形状
- 添加光泽效果
- 可选装饰元素（小星星、小爱心）

## Risks / Trade-offs

### 风险1: 动画过多导致分心
**缓解**: 练习页面减少背景动画，聚焦键盘区域

### 风险2: 字体加载延迟
**缓解**: 使用 `font-display: swap` + 预加载关键字体

### 风险3: 低端设备性能问题
**缓解**: 
- 使用 `prefers-reduced-motion` 媒体查询
- 复杂动画仅在支持的设备上启用

## Migration Plan
1. 更新 `main.css` 全局样式变量和基础样式
2. 逐个更新组件样式（NavBar → HomePage → ExercisePage → 其他页面）
3. 添加背景动画效果
4. 测试各页面视觉效果和性能
5. 修复任何样式冲突

## Open Questions
- 是否需要添加暗色模式支持？（建议暂不添加，儿童应用通常不需要）
- 是否需要添加更多的音效配合视觉反馈？（超出本次范围）
